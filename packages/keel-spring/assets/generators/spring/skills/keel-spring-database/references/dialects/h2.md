# H2 (`database: "h2"`)

Lo que build dejó: `com.h2database:h2` como `runtimeOnly`, **sin contenedor**
(ni entrada en `infra/docker-compose.yaml`, ni `reset-db.sh` para la BD), URL
`jdbc:h2:mem:<servicio_snake>;MODE=LEGACY;DB_CLOSE_DELAY=-1`.

H2 como base del stack es una elección para demos/prototipos: todo vive en
memoria del proceso.

## Consecuencias operativas

- **Los datos mueren con la app**: cada reinicio de `bootRun` parte de esquema
  recién creado (`ddl-auto: update` sobre BD vacía). El «reset de datos entre
  flujos» de la validación es, simplemente, reiniciar la aplicación.
- No hay validación de infraestructura para la BD (`validate-infra.sh` no la
  incluye): si la app arranca, la BD está.
- `DB_CLOSE_DELAY=-1` mantiene la BD viva aunque se cierren todas las
  conexiones; sin él, el pool podría vaciarse y perder el esquema a mitad de
  ejecución.
- production con H2 en memoria no es un despliegue real: si este diseño va a
  producción, el stack debe cambiar de BD (borra `keel-stack.json` y
  re-ejecuta `keel-spring build --force`).

## Modos de compatibilidad

- La URL de build usa `MODE=LEGACY` (comportamiento H2 1.x, el más permisivo).
- El **perfil test** (para cualquier BD del stack, también H2) usa otra URL:
  `jdbc:h2:mem:testdb;MODE=PostgreSQL;DATABASE_TO_LOWER=TRUE` — no asumas que
  runtime y test se comportan igual (casing de identificadores y funciones
  cambian entre modos).
- Si una query necesita una función específica de un modo, decláralo en la URL
  conscientemente; no mezcles modos entre perfiles sin motivo.

## Tipos

- `uuid`, `json` (limitado, sin operadores de documento), `clob`/`text`:
  suficientes para prototipos; nada de índices GIN ni columnas generadas
  complejas.
- Secuencias nativas: `GenerationType.SEQUENCE` y batching funcionan.

## Inspección

Sin CLI externa; para mirar datos usa la consola H2 solo si la habilitas
explícitamente en local (`spring.h2.console.enabled: true`,
ruta `/h2-console`) — **jamás en develop/production** (es un shell SQL
abierto). Alternativa sin consola: un test `@SpringBootTest` que consulte el
repositorio.
